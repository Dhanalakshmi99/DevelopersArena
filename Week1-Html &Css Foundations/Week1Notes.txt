1.HTML5 structure and semantic elements:

A basic HTML5 page follows a standard structure that defines how content is organized and interpreted by browsers.

Example:

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My HTML5 Page</title>
  </head>
  <body>
    <header>
      <h1>Website Title</h1>
      <nav>
        <ul>
          <li><a href="#">Home</a></li>
          <li><a href="#">About</a></li>
          <li><a href="#">Contact</a></li>
        </ul>
      </nav>
    </header>

    <main>
      <article>
        <h2>Article Title</h2>
        <p>This is the main content of the article.</p>
      </article>

      <section>
        <h2>Related Topics</h2>
        <p>Additional information here.</p>
      </section>

      <aside>
        <h3>Sidebar</h3>
        <p>Links or ads can go here.</p>
      </aside>
    </main>

    <footer>
      <p>&copy; 2025 My Website</p>
    </footer>
  </body>
</html>


üß© Semantic Elements in HTML5:

Semantic elements describe their meaning both to the browser and the developer, improving accessibility, SEO, and readability.

| Element        | Description                         | Typical Use                                       |
| -------------- | ----------------------------------- | ------------------------------------------------- |
| `<header>`     | Introductory content or navigation  | Top of a page or section                          |
| `<nav>`        | Group of navigation links           | Main site or in-page navigation                   |
| `<main>`       | Main content unique to the document | Only one per page                                 |
| `<section>`    | Thematic grouping of content        | Divides main content into topics                  |
| `<article>`    | Independent, self-contained content | Blog posts, news, or forum entries                |
| `<aside>`      | Tangential or supplementary content | Sidebars, pull quotes, ads                        |
| `<footer>`     | Footer for a page or section        | Copyright info, links, contact                    |
| `<figure>`     | Self-contained media                | Images, charts, code samples                      |
| `<figcaption>` | Caption for `<figure>`              | Describes the figure content                      |
| `<time>`       | Represents time or date             | `<time datetime="2025-11-13">Nov 13, 2025</time>` |
| `<mark>`       | Highlights text                     | Marking search results                            |
| `<address>`    | Contact information                 | Author or organization info                       |

‚úÖ Benefits of Using Semantic HTML5 Elements

Improves Accessibility ‚Äî Screen readers and assistive technologies understand content structure.

Better SEO ‚Äî Search engines can interpret content meaning more accurately.

Cleaner Code ‚Äî Easier to maintain and read.

Consistent Styling ‚Äî Easier to target specific content areas with CSS.

2.CSS basics: selectors, properties, values:

‚úÖ 1. Selectors

Selectors tell CSS which HTML elements to style.

Examples of selectors:
Element selector
p {
  color: blue;
}


Selects all <p> elements.

Class selector
.button {
  background: red;
}


Selects any element with class="button".

ID selector
#header {
  font-size: 24px;
}


Selects the element with id="header".

Attribute selector
input[type="text"] {
  border: 1px solid gray;
}

Descendant selector
nav ul li {
  list-style: none;
}

‚úÖ 2. Properties

Properties are the characteristics you want to change.

Examples of CSS properties:
color: red;
font-size: 20px;
margin: 10px;
background-color: yellow;
border: 2px solid black;
width: 50%;


Common examples:

color

background-color

padding

margin

font-size

display

‚úÖ 3. Values

Values tell CSS how to apply the property.

Examples of values:
color: blue;                /* value = blue */
font-size: 16px;            /* value = 16px */
margin: 20px auto;          /* values = 20px, auto */
display: flex;              /* value = flex */
background-color: #ff0000;  /* value = #ff0000 */

‚≠ê Putting it all together
/* selector */   /* property */   /* value */
p {
  color: green;
  font-size: 18px;
}

.button {
  background-color: purple;
  padding: 10px;
}

3.Box model: margin, padding, border:

Definition:

The CSS box model describes how every HTML element is structured as a rectangular box made of:

Content ‚Äì the actual text or image.

Padding ‚Äì space inside the box around the content.

Border ‚Äì the line surrounding the padding and content.

Margin ‚Äì space outside the border separating the element from others.

Use:

Designers use the box model to control spacing, alignment, and layout. It ensures consistent spacing across elements.

Example:

<div class="card">Hello</div>

<style>
.card {
  padding: 20px;   /* space inside */
  border: 2px solid black; /* border line */
  margin: 15px;    /* space outside */
}
</style>

4.CSS positioning and layout fundamentals:

Definition:

These are the techniques used to control where elements appear on a page. Key positioning types include:

static ‚Äì default, follows normal flow.

relative ‚Äì adjusts an element relative to its normal position.

absolute ‚Äì positioned relative to the nearest positioned ancestor.

fixed ‚Äì stays in the same place even when scrolling.

sticky ‚Äì toggles between relative and fixed based on scroll.

Other layout tools include flexbox, grid, and float.

Use:

Layout systems help create structured interfaces, navigation bars, columns, responsive grids, and more.

Example: Flexbox layout

<div class="container">
  <div>Box 1</div>
  <div>Box 2</div>
  <div>Box 3</div>
</div>

<style>
.container {
  display: flex;
  justify-content: space-between;
}
</style>

Example: Positioning
<div class="box">I'm fixed!</div>

<style>
.box {
  position: fixed;
  top: 0;
  right: 0;
}
</style>

5.Responsive design principles:

Definition:

Responsive design ensures that websites adapt smoothly to different screen sizes (phone, tablet, desktop). It involves flexible layouts, images, and CSS breakpoints.

Use:

Ensures accessibility and usability on all devices, improves user experience, and is essential for modern web development.

Key Principles

Fluid grids (percent-based widths)

Media queries (apply styles based on screen size)

Flexible images and content

Mobile-first design

Example:

/* Mobile first */
.container {
  width: 100%;
}

/* For screens wider than 768px */
@media (min-width: 768px) {
  .container {
    width: 50%;
  }
}

‚úÖ 6. Web Accessibility Basics:

Definition:

Web accessibility ensures that websites are usable by everyone, including people with disabilities (visual, motor, cognitive, auditory).

Use:

Accessibility improves usability, meets legal standards (WCAG/ADA), and makes websites more inclusive.

Core Concepts

Alt text for images

Keyboard navigation support

Color contrast compliance

Semantic HTML for screen readers

Example:

<img src="dog.jpg" alt="Golden retriever playing in the park">

<button aria-label="Close menu">‚úï</button>

<nav>
  <ul>
    <li><a href="#home">Home</a></li>
  </ul>
</nav>
